- **Error Message Style:** Format "[tool_name] Failed to X: reason" + context + suggestion

## Research Findings

From 02-RESEARCH.md:
- `GET /pullrequests` - List PRs with `state` query param (OPEN/MERGED/DECLINED)
- `GET /pullrequests/{id}` - Get single PR details
- State parameter: normalize lowercase input ('open', 'merged', 'declined') to API values ('OPEN', 'MERGED', 'DECLINED')
- Use `pagelen=20` parameter to get 20 most recent
- Safer nested access: `pr.get('source', {}).get('branch', {}).get('name', 'unknown')`
